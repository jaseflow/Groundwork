Thankfully with the introduction of HTML5 we don't have to write anywhere near as much as we use to have to but that still doesn't mean this part is any less important.

The Doctype

<!DOCTYPE HTML>

Surely everyone is now working with this DOCTYPE and has been for a long time. This is supported by all browsers that mean anything (all it does is trigger standards mode) and is a million times shorter than any previous DOCTYPE.

The Charset

<meta charset="UTF-8">

This attribute declares the character encoding used for the page. UTF-8 is the encouraged character encoding. The META element must be within the <head> and within the 512 first bytes of the page. That will surely come in use next pub night.

The Viewport

<meta name="viewport" content="width=device-width, initial-scale=1.0>

This is a topic of great discussion. This META tag is a key ingredient to building responsive websites, controlling the way the phone reacts to different device resolutions (mobiles, tablets etc). width=device-width tells the browser to assume that the page is as wide as the device. initial-scale=1.0" tells the browser not to zoom in or out and not make assumptions that the website is built for larger screens. You could set a maximum-scale=1.0 if you would like to stop the user from zooming on the phone but there shouldn't be any reason you need to do this.

NB: Remember to always import your Javascript files after your stylesheets to increase performance and stop any flashes of unstyled content (FOUC).
