Thankfully with the introduction of HTML5 we don't have to write anywhere near as much as we use to have to but that still doesn't mean this part is any less important.

The Doctype

<!DOCTYPE HTML>

Surely everyone is now working with this DOCTYPE and has been for a long time. This is supported by all browsers that mean anything (all it does is trigger standards mode) and is a million times shorter than any previous DOCTYPE.

Remember to always import your Javascript files after your stylesheets to increase performance and stop any flashes of unstyled content (FOUC).

The Viewport

<meta name="viewport" content="width=device-width, initial-scale=1.0>

The viewport tag is a key ingredient to building responsive websites. It controls the way the website reacts to different device resolutions (mobiles, tablets etc). width=device-width tells the browser to assume that the page is as wide as the device. initial-scale=1.0" tells the browser not to zoom in or out and not make assumptions that the website is built for larger screens. You could set a maximum-scale=1.0 if you would like to stop the user from zooming on the phone but there shouldn't be any reason you need to do this. If you want to more I suggest read the MDN page on META tags.
